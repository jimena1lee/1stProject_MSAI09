# 통합 데이터 발생건수 회귀 분석 파이프라인

> **파일**: `regression_integrated.ipynb`
> **데이터**: `seongnam_train.csv` 단일 파일 (학습 / 검증 / 테스트 모두 사용)
> **레이블**: `발생건수`

---

## 1. 데이터 개요

### 데이터 구성 및 분할 전략

`seongnam_train.csv` (110건)을 2단계로 나눠 학습·검증·테스트 세 단계를 모두 수행한다.

```
seongnam_train.csv (110건)
│
├─ 1차 분할 (80 / 20)
│   ├── trainval (88건)
│   │     └─ 2차 분할 (80 / 20)
│   │           ├── train  (70건, ≈64%)  ← scaler.fit + model.fit
│   │           └── val    (18건, ≈16%)  ← 과적합 모니터링
│   └── test   (22건, ≈20%)             ← 최종 성능 평가
```

> scaler는 학습셋(70건)에서만 `fit`하고, 검증·테스트에는 `transform`만 적용하여 데이터 누수를 방지한다.

### 레이블 분포 (`발생건수`)

| 구분 | 0건 | 3건 | 합계 |
|---|---|---|---|
| 전체 | 104 (94.5%) | 6 (5.5%) | 110 |
| Train | 66 (94.3%) | 4 (5.7%) | 70 |
| Val | 17 (94.4%) | 1 (5.6%) | 18 |
| Test | 21 (95.5%) | 1 (4.5%) | 22 |

> 발생건수는 `0` 또는 `3` 두 값만 존재하는 **극도로 불균형한 이진 분포**이다.

### 입력 피처 (10개)

Drop 컬럼: `시설물명`, `위도`, `경도`, `structure_risk` (파생 변수)

| 피처명 | 설명 |
|---|---|
| 도로안전표지 | 도로 안전 표지판 수 |
| 도로적색표면 | 적색 노면 표시 수 |
| 무단횡단방지펜스 | 무단횡단 방지 펜스 수 |
| 무인교통단속카메라 | 단속 카메라 수 |
| 보호구역표지판 | 어린이 보호구역 표지판 수 |
| 생활안전CCTV | 생활 안전 CCTV 수 |
| 신호등 | 신호등 수 |
| 옐로카펫 | 옐로카펫 수 |
| 횡단보도 | 횡단보도 수 |
| 어린이 비율(%) | 해당 구역 어린이 인구 비율 |

## 5. 종합 평가

### 성능 요약

| 지표 | Train | Val | Test |
|---|---|---|---|
| R² | 0.2343 | −0.1060 | −0.1826 |
| MAE | 0.3744 | 0.4757 | 0.4766 |
| RMSE | 0.6093 | 0.7227 | 0.6796 |
| Accuracy | − | 0.8333 | 0.8182 |
| Recall (macro) | − | 0.9118 | 0.9048 |
| F1 (macro) | − | 0.6516 | 0.6140 |

### 한계 및 해석

1. **레이블 이진성 문제**: 발생건수가 0 또는 3 두 값만 존재해 연속형 회귀 모델에 구조적으로 부적합. 로지스틱 회귀 또는 이진 분류 모델이 더 적합하다.

2. **극심한 클래스 불균형**: 전체 110건 중 발생건수 > 0인 샘플이 6건(5.5%)에 불과. 테스트셋에도 양성 샘플이 1건뿐이라 소수 클래스 평가가 불안정하다.

3. **낮은 선형 설명력**: 피처-레이블 간 최대 상관계수 약 0.31로 선형 관계가 약하다. 랜덤포레스트, XGBoost 등 비선형 모델 또는 SMOTE를 통한 오버샘플링 적용을 고려할 필요가 있다.

4. **잔차 비정규성**: 검증/테스트 모두 Shapiro-Wilk p < 0.05로 선형회귀의 잔차 정규성 가정을 위반한다. 레이블 분포 특성에서 비롯된 구조적 한계이다.

5. **분류 관점 성능**: Accuracy 81~83%는 다수 클래스(0건)를 맞추는 trivial 분류에 가깝다. Macro F1 = 0.61~0.65가 실질적인 성능 지표이며, 소수 클래스 탐지에는 한계가 있다.
