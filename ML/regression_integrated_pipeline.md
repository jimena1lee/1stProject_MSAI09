# 통합 데이터 발생건수 회귀 분석 파이프라인

> **파일**: `regression_integrated.ipynb`
> **데이터**: `seongnam_train.csv` (학습/검증) · `seongnam_test.csv` (최종 테스트)
> **레이블**: `발생건수`

---

## 1. 데이터 개요

### 데이터 구성

| Train (`seongnam_train.csv`) | 110 | 15 |
| Test (`seongnam_test.csv`) | 32 | 15 |

### 레이블 분포 (`발생건수`)

> 통합 데이터(시설물 정보 + 구조 위험 점수)를 결합한 성남시 스쿨존 단위 데이터이다.

### 입력 피처 (10개)

Drop된 컬럼: `시설물명`, `위도`, `경도`, `structure_risk` (파생 변수)

### STEP 3 | 모델 학습

**모델**: `sklearn.linear_model.LinearRegression` (다중선형회귀)

**전처리**: `StandardScaler` (평균 0, 분산 1 정규화)
→ 스케일이 다른 피처 간 회귀계수를 공정하게 비교하기 위해 적용

**데이터 분할 전략**:

```
seongnam_train.csv (110건)
  ├── 학습셋 (88건, 80%)  → scaler.fit_transform + model.fit
  └── 검증셋 (22건, 20%)  → scaler.transform + model.predict

seongnam_test.csv (32건)
  └── 최종 테스트셋       → scaler.transform + model.predict
```

> scaler는 학습셋에서만 `fit`하고, 검증/테스트에는 `transform`만 적용해 데이터 누수를 방지.

## 3. 검증 결과 (Validation Set, 22건)

### 3-1. 회귀 지표

| 지표 | 학습셋 | 검증셋 |
|---|---|---|
| R² | 0.1990 | **−0.1817** |
| MAE | 0.3706 | 0.4859 |
| RMSE | 0.6216 | 0.6793 |

- **Train R² = 0.199**: 학습셋에서도 설명력이 낮음 (약 20%만 설명)
- **Val R² = −0.182**: 검증셋에서 음수 → **모델이 평균 예측보다 오히려 나쁜 성능**
- 과적합 지수 (Train R² − Val R²) = **0.381** → 과적합 발생

### 3-2. 분류 지표 (이진 이산화: 0.5 기준)

| 지표 | 검증셋 |
|---|---|
| Accuracy | 0.7727 |
| Recall (macro) | 0.8810 |
| F1-Score (macro) | 0.5753 |

```
              precision    recall  f1-score   support

    없음(0건)       1.00      0.76      0.86        21
```

### 3-3. 잔차 분석 (검증셋)

| 항목 | 값 |
|---|---|
| Shapiro-Wilk W | 0.8420 |

## 5. 종합 평가 및 한계

### 성능 요약

| 지표 | 학습 | 검증 | 테스트 |
|---|---|---|---|
| R² | 0.1990 | −0.1817 | 0.0821 |
| MAE | 0.3706 | 0.4859 | 0.3713 |
| RMSE | 0.6216 | 0.6793 | 0.6957 |
| Accuracy | − | 0.7727 | 0.9062 |
| F1 (macro) | − | 0.5753 | 0.6746 |

### 한계 및 해석

1. **레이블 이진성 문제**: 발생건수가 0 또는 3 두 값만 존재해 연속형 회귀 모델이 적합하지 않은 데이터 구조. 로지스틱 회귀나 의사결정나무 등 분류 모델이 더 적합할 수 있음.

2. **극심한 클래스 불균형**: 발생건수 > 0인 샘플이 train 6건, test 2건에 불과해 소수 클래스 학습이 매우 어려움. SMOTE 등 오버샘플링 또는 클래스 가중치 적용 고려 필요.

3. **낮은 선형 설명력**: 피처와 레이블 간 최대 상관계수가 약 0.31로 선형 관계가 약함. 비선형 모델(랜덤포레스트, XGBoost 등) 시도가 필요.

4. **잔차 비정규성**: 레이블 분포 특성상 선형회귀의 잔차 정규성 가정이 성립하지 않음.

5. **긍정적 측면**: Accuracy 90.6%로 보이나 이는 다수 클래스(0건)를 맞히는 trivial 분류에 가까움. Macro F1 = 0.67이 더 실질적인 성능 지표.
