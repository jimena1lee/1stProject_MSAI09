## 5. 모델별 특성 분석

### Logistic Regression
- Accuracy(0.824)와 Macro F1(0.813)이 **5개 모델 중 가장 높음**
- AUC-ROC는 0.757로 가장 낮지만, 발생 클래스 Recall(0.714)이 XGBoost와 함께 높음
- 선형 경계로도 어느 정도 판별 가능한 구조임을 시사

### SVM (RBF)
- 비선형 커널로 AUC-ROC 0.771 달성
- FN=4로 사고 발생 지점을 놓치는 비율이 높음
- Recall이 0.429로 낮아 실무적 탐지 성능 한계

### Random Forest
- AUC-ROC 0.800으로 확률 판별력은 준수하나
- 실제 예측(Accuracy=0.647, F1=0.50)은 다소 낮음 → max_depth=5 제약의 영향
- **피처 중요도 해석에 주로 활용**

### XGBoost
- AUC-ROC 0.814로 2위, 클래스1 Recall(0.714)이 높아 발생 지점 탐지에 유리
- FN=2 (놓친 발생 샘플 가장 적음) → 실무 관점에서 LR과 함께 우수
- 클래스1 F1 0.667로 균형 잡힌 성능

### KNN (최적 모델)
- AUC-ROC **0.857**로 1위 선정
- FP=0: **미발생을 발생으로 오분류한 경우 없음** (Precision=1.000)
- 단, FN=4로 발생 지점 4곳을 미발생으로 분류 (Recall=0.429)
- → **"발생 예측 시 확실히 맞추지만, 발생을 놓칠 수 있는"** 보수적 분류기

## 8. 종합 결론

### 모델 선택
> **KNN(K=5)** 이 val AUC-ROC 기준 최적 모델 (AUC=0.857 → Test AUC=0.925)

- AUC-ROC 관점: KNN > XGBoost > RF > SVM > Logistic Regression
- Accuracy 관점: Logistic Regression > KNN = XGBoost > SVM > Random Forest
- 클래스1 F1 관점: Logistic Regression > XGBoost > KNN > SVM > Random Forest

**목적에 따른 모델 선택 가이드**

| 목적 | 추천 모델 | 이유 |
|---|---|---|
| 위험 구역 스크리닝 (놓치면 안 됨) | **XGBoost** | Recall 0.714, FN=2로 미탐 최소 |
| 발생 확률 순위화 (전체 판별력) | **KNN** | AUC-ROC 0.857 최고 |
| 오탐 최소화 (확실한 것만 발생 예측) | **KNN** | Precision 1.000, FP=0 |
| 계수 해석이 필요한 경우 | **Logistic Regression** | 명시적 log-odds 계수 제공 |

### 주요 인사이트
1. **인구 특성이 가장 중요한 판별 변수**: 어린이 비율(%)이 LR 계수 기준 가장 강한 음의 예측력 → 어린이가 많은 조용한 주거지역보다 통학로·상업지역에서 사고 발생
2. **CCTV·단속카메라의 역인과 관계**: 위험 구역을 나타내는 지표이지, 위험을 유발하는 요인이 아님 → 정책 해석 시 주의 필요
3. **도로적색표면·옐로카펫의 보호 효과**: 음의 계수 → 시각적 주의 환기 시설의 실질적 효과 확인

### 한계 및 개선 방향
- **소표본 문제**: Val=17, Test=18으로 지표 변동성이 큼 — 교차검증(CV) 도입 권장
- **KNN의 이웃 수 최적화**: k=5 고정 → GridSearch로 최적 k 탐색 가능
- **클래스 불균형 심화 시**: SMOTE 등 오버샘플링 기법 적용 고려
- **공간적 자기상관 미반영**: 인근 보호구역 간 영향 관계 → GNN 등 그래프 기반 모델 도입 여지
