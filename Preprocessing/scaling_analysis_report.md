## 3. 스케일링 방법 결정

### 3-1. 스케일러 비교 검토

| 스케일러 | 특징 | 이 데이터에서의 판단 |
|---------|------|------------------|
| MinMaxScaler | 0\~1 범위 압축, 이상치에 민감 | 이상치가 max를 결정해 나머지 값 압축 우려 |
| StandardScaler | 평균 0, 표준편차 1, 이상치에 상대적으로 강함 | 이상치가 적어 안정적, 모델 범용성 높음 |
| RobustScaler | 중앙값/IQR 기준, 이상치에 가장 강함 | 이상치가 충분히 적어 필수적이지 않음 |
| Log(x+1) + Standard | 비정규 분포를 먼저 완화 후 표준화 | 강하게 치우친 feature에 적합 |

### 3-2. 최종 결정

**조건**
- 신경망 미사용 (MinMaxScaler의 핵심 장점 불필요)
- 이상치 2\~3개 수준 (MinMaxScaler보다 StandardScaler가 안전)
- 강하게 치우친 feature 존재 (Log 변환 선행 필요)

**적용 전략**

| 대상 feature | skewness | 변환 방식 |
|-------------|---------|---------|
| 무인교통단속카메라 | 1.44 | Log(x+1) → StandardScaler |
| 보호구역표지판 | 2.00 | Log(x+1) → StandardScaler |
| 생활안전CCTV | 1.41 | Log(x+1) → StandardScaler |
| 옐로카펫 | 1.01 | Log(x+1) → StandardScaler |
| 나머지 11개 | < 1.0 | StandardScaler |
