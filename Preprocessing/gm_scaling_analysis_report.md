## 3. 스케일링 방법 결정

### 3-1. 스케일러 비교 검토

| 스케일러 | 특징 | 이 데이터에서의 판단 |
|---------|------|------------------|
| MinMaxScaler | 0\~1 범위 압축, 이상치에 민감 | 도로안전표지처럼 극단적 0 편중 시 범위 압축 우려 |
| StandardScaler | 평균 0, 표준편차 1, 이상치에 상대적으로 강함 | 소표본에서 안정적, 범용성 높음 |
| RobustScaler | 중앙값/IQR 기준, 이상치에 가장 강함 | 이상치 수준이 과도하지 않아 필수적이지 않음 |
| Log(x+1) + Standard | 비정규 분포를 먼저 완화 후 표준화 | 강하게 치우친 feature에 선행 적용 |

### 3-2. 최종 결정

**적용 전략**

| 대상 feature | skewness | 변환 방식 |
|-------------|---------|---------|
| 무인교통단속카메라 | 0.38 | Log(x+1) → StandardScaler |
| 보호구역표지판 | 1.37 | Log(x+1) → StandardScaler |
| 생활안전CCTV | 0.54 | Log(x+1) → StandardScaler |
| 옐로카펫 | 2.00 | Log(x+1) → StandardScaler |
| 도로안전표지 | 3.23 | StandardScaler |
| 도로적색표면 | 1.00 | StandardScaler |
| 무단횡단방지펜스 | 1.07 | StandardScaler |
| 신호등 | 1.64 | StandardScaler |
| 횡단보도 | 1.00 | StandardScaler |
| 어린이 비율(%) | -1.23 | StandardScaler |

> ⚠️ **주의**: Log 변환 대상 선정 기준이 `skewness > 1`로 명시되어 있으나, 실제 적용 시 `무인교통단속카메라(0.38)`, `생활안전CCTV(0.54)`는 기준 미달임에도 Log 변환에 포함됨. 반면 `도로안전표지(3.23)`, `신호등(1.64)` 등 기준 초과 항목은 StandardScaler만 적용됨 → 추후 정책 통일 필요
